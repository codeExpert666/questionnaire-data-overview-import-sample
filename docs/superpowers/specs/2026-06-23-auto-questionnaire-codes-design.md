# Auto Questionnaire Codes Design

## Context

`pq_product.product_code` 和 `pq_feature.feature_code` 是 Excel 模板、导入解析和外部配置使用的稳定编码。当前创建产品和特性时编码必填，创建后不可修改。

本次调整只放宽创建接口：调用方未传入编码或只传入空白字符串时，由服务自动生成稳定编码。已传入的编码继续按现有规则去空白、校验字符集和查重。

## Design

产品创建时，若 `productCode` 为空，服务先使用临时编码插入 `pq_product`，拿到数据库自增 `id` 后再把编码更新为 `P{id}`，最后按主键回读完整记录。

特性创建时，若 `featureCode` 为空，服务先使用临时编码插入 `pq_feature`，拿到数据库自增 `id` 后再把编码更新为 `F{id}`，最后按主键回读完整记录。

临时编码使用合法字符前缀并带随机后缀，避免与已有人工编码冲突。最终编码仍受数据库唯一索引保护；如果最终编码已被人工占用，服务返回现有“编码已存在”业务错误。

## Data Flow

1. 创建请求进入 Service。
2. 先规范化名称、排序号和状态。
3. 编码非空时走原有创建流程。
4. 编码为空时生成临时编码并插入。
5. Mapper 增加只更新编码的方法，按自增 `id` 写入 `P{id}` 或 `F{id}`。
6. 更新成功后递增缓存版本，并回读响应。

## Scope

不改变更新接口，不改变 Excel 导入解析，不改变模板下载结构，不新增数据库表或配置项。

## Testing

新增服务层测试覆盖：

- 产品创建未传 `productCode` 时写入最终编码 `P{id}`。
- 特性创建未传 `featureCode` 时写入最终编码 `F{id}`。
- 已传编码的旧行为继续保留。
